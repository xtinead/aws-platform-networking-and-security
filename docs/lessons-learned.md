# Lessons Learned

Designing AWS networking and security foundations reinforced several important platform engineering principles.

## Key Lessons

- cloud networking should be designed intentionally, not implicitly
- ingress and runtime boundaries should be separated
- least privilege access patterns matter across both IAM and networking
- public and private subnet segmentation remains foundational
- blast-radius control should be visible in both diagrams and implementation

## Outcome

This repository demonstrates how AWS network and security architecture supports the broader platform ecosystem.
